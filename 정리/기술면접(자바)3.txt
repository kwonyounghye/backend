single thread(순서)
	작업단위 1개
multi thread(속도)
	작업단위 2개 이상

Thread 단점: 교착상태에 빠져서 꼬일 수 있다.

protected 같은 패키지 + 하위 클래스에서 접근 가능

singleton 싱글톤: 인스턴스를 1개 만듦

소켓통신(TCP: 연결형 서비스-정확(ex. 인터넷)/UDP:연결이 끊어진 상태에서 일방적으로 데이터를 보냄-신뢰성이 낮음, 속도는 빠름, 연결이 끊어질 수 있음(ex. 방송, 유튜브 스트리밍))

Stack(LIFO:Last In First Out 마지막에 들어간 게 먼저 나감)
입력 push
출력 pop

Queue

primary key: 인덱스

객체지향
	캡슐화 : 변수는 private으로 숨기고(자료 보호) 다른 것은 public
	상속 : 코드 재사용
	추상화 : 추상클래스, 인터페이스 - 표준 만듦
	다형성 : 프로그램을 편리하게 짤 수 있다.

list [] 변경가능
tuple() 변경안됨
